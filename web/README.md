## Что такое Stride

Stride — внутренняя платформа для школ, которая объединяет наблюдения педагогов, профили учеников и аналитику в одном месте. Система помогает классным руководителям и администрации отслеживать учеников в зоне риска и принимать решения на основе данных.

## Страницы и функции (Неделя 1)

### `/` — Главная (Dashboard)
- 4 карточки статистики с **реальными трендами**: Ученики (+N за месяц), Ученики в зоне риска, Наблюдения (+N за неделю), Тревожные наблюдения (+N за неделю)
- Лента последних наблюдений с поиском и фильтром по классу, фото учеников
- Блок тревожных наблюдений (для кл. руководителей — только свой класс)
- Блок учеников без наблюдений 14+ дней
- AI Выводы (для admin/deputy/manager)
- Смена учебного года (только admin/manager) с подтверждением

### `/students` — Список учеников
- Группировка по классам, сортировка: 12→9 по убыванию, А→В по возрастанию
- Для каждого класса: общее число наблюдений, тревожных наблюдений, учеников в зоне риска
- Для каждого ученика: число наблюдений, тревожных, статус (Норма / В зоне риска красным)
- Клик по строке класса = раскрыть/свернуть, клик по аватарке/ФИО = профиль ученика
- Поиск по имени

### `/students/[id]` — Профиль ученика
- Фото, ФИО, класс, классный руководитель, статус, теги интересов
- Контакты родителей
- Вкладки: Наблюдения | AI Сводка | Книга ученика
- Кнопка «Добавить наблюдение» → `/observations/new?student=ID`
- Редактирование профиля (модалка)

### `/students/new` — Добавить учеников
- **3 зоны импорта Excel**: Анкета учителя + Анкета ученика + Анкета родителя
- Кнопка «Создать профили» активна только если загружены **все три** файла
- Данные объединяются по ФИО ученика, классы создаются автоматически
- Подсказки по колонкам для каждой анкеты

### `/staff` — Сотрудники
- Таблица сотрудников с ролями
- Копирование пароля (показывает «Скопировано» на 2 сек)

### `/staff/new` — Добавить сотрудника
- Форма добавления

### `/observations/new` — Добавить наблюдение
- Поддерживает `?student=ID` — предзаполняет ученика и класс

### `/student-book` — Книга ученика
- Поиск по ФИО (комбобокс с вводом)
- Генерация PDF-отчёта

### `/analytics` — Аналитика школы
- Доступна: admin, deputy, manager
- 4 карточки: Всего учеников, В зоне риска, Наблюдений, **Тревожных наблюдений**
- Графики: Причины ухода, Любимые предметы, Популярные виды спорта
- Блок ИИ-анализа: сводный баннер + 7 карточек паттернов/трендов + методология

## База данных (Supabase)

**Проект:** `wbfegsgixpsoratvmlhp` (Stride)

### Ключевые таблицы
- `schools` — school_year, name, logo_url
- `users` — role, class_id, school_id
- `classes` — grade, letter, name, school_id
- `students` — full_name, class_id, risk_level, goals, interests, photo_url, status
- `observations` — content, category, is_alert, student_id, author_id

### RLS политики
- Сотрудники видят данные только своей школы (`get_my_school_id()`)
- UPDATE наблюдений: автор ИЛИ admin/deputy/manager ИЛИ кл. руководитель (только своего класса)
- Смена учебного года: только через `service_role` (admin client)

### RPC функции
- `get_students_no_recent_obs(p_school_id)` — ученики без наблюдений 14+ дней
- `increment_class_grades(p_school_id)` — повысить параллель всех классов
- `decrement_class_grades(p_school_id)` — откатить параллель

## Как показывать клиенту

### Демо-аккаунты для показа
Создать в Supabase → Authentication → Users:
- **Директор**: `director@school.com` / роль `admin`
- **Кл. руководитель**: `teacher@school.com` / роль `class_teacher` + привязать класс
- **Учитель**: `staff@school.com` / роль `teacher`

### Ключевые сценарии демо
1. **Войти как директор** → показать главную с трендами и тревожными наблюдениями
2. **Список учеников** → раскрыть класс, показать статистику по классу и ученикам
3. **Профиль ученика** → вкладка AI Сводка, кнопка «Добавить наблюдение»
4. **Войти как кл. руководитель** → тревожные наблюдения только своего класса
5. **Аналитика** → ИИ-анализ с карточками паттернов
6. **Добавить учеников** → показать 3 зоны импорта Excel

## Неделя 2 — Внедрение AI 🚀

### Планируемые функции

| Функция | Описание |
|---|---|
| **Реальный AI-анализ** | Подключить Claude API для анализа наблюдений по школе |
| **AI Сводка ученика** | Авто-генерация сводки по наблюдениям конкретного ученика |
| **Умные рекомендации** | Персональные советы педагогу на основе паттернов |
| **Предсказание риска** | Модель определения учеников в зоне риска по наблюдениям |
| **Автокатегоризация** | Автоматически определять категорию наблюдения (AI, не педагог) |
| **Поиск по смыслу** | Семантический поиск по наблюдениям |
| **Еженедельный дайджест** | Авто-отчёт директору: что произошло за неделю |

### Технический план
- Supabase Edge Functions для вызовов Claude API
- Модель: `claude-sonnet-4-6` (баланс скорость/качество)
- Хранение AI-ответов в `ai_summaries` таблице с кешированием
- Streaming для длинных ответов (AI Сводка ученика)
